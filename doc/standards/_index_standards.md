# Индекс пакета стандартов. Версия 4.9.0. 2026-07-21

Точка входа в пакет стандартов. Этот файл тегают AI-агенту, чтобы он определил, какие документы пакета релевантны текущей задаче. Пакет состоит из основного стандарта и дополнений: каждое дополнение применяется поверх `project_standards.md` для своего класса проектов. Агент выбирает только те документы, которые относятся к задаче, и не загружает пакет целиком без необходимости.

Пакет имеет единую версию (см. раздел "Политика версионирования"). Версия во всех документах пакета, в этом индексе, а также в `AGENTS.md`, `CLAUDE.md`, `README.md` проекта - одна и та же.

## Обязательность применения

Тегание этого файла (`@doc/standards/_index_standards.md`) означает **обязательное применение** стандартов пакета, а не просто ознакомление с ними. Агент:

1. Определяет релевантные документы по карте маршрутизации
2. Загружает указанные документы и применяет их требования к текущей задаче
3. Соблюдает все правила и ограничения, описанные в стандартах
4. Проверяет соответствие результата стандартам перед завершением работы
5. Передает требования стандартов субагентам; субагенты обязаны применять стандарты в том же объеме, что и основной агент

Стандарты применяются всегда при работе с проектом, независимо от того, был ли явно тегнут этот файл. Тегание лишь указывает агенту на необходимость явно загрузить и применить пакет.

Субагенты (запущенные через task) применяют стандарты пакета наравне с основным агентом.

## Политика версионирования

Пакет стандартов имеет единую версию - семантическое версионирование (SemVer) `MAJOR.MINOR.PATCH`. Версия едина для всего пакета: индивидуальные версии документов упразднены.

Формат заголовка каждого документа: `# <Имя документа>. Версия X.Y.Z. YYYY-MM-DD`.

**Правила изменения версии:**

- **PATCH** (X.Y.Z -> X.Y.Z+1) - уточнения формулировок, исправление опечаток, форматирование, правки примеров
- **MINOR** (X.Y.Z -> X.Y+1.0) - новые разделы, новые правила, новые требования в рамках существующих документов
- **MAJOR** (X.Y.Z -> X+1.0.0) - добавление или удаление документов пакета, несовместимые изменения структуры, смена модели (например, переход на единую версию пакета)

**Bump версии пакета - только по явному запросу пользователя** («подними версию», «bump» и аналоги). Агент при правке содержимого `doc/standards/` **не** поднимает версию сам и не требует bump как обязательный шаг завершения задачи.

При запросе bump:

1. Поднять версию пакета по правилам PATCH/MINOR/MAJOR выше (если пользователь не указал уровень)
2. Обновить дату в заголовках
3. Поставить новую версию во ВСЕ локации (одно значение):
   - `doc/standards/_index_standards.md` (этот файл)
   - все документы `doc/standards/*.md` (заголовки)
   - `AGENTS.md` (строка `Standard version:`)
   - `CLAUDE.md` (строка `Standard version:`)
   - `README.md` (строка `Соответствует стандарту:`)

Реестром версии служит этот индекс. Когда версии проставлены, во всех перечисленных локациях они должны совпадать. Правка содержимого стандартов без bump номера допустима; актуализация реестра и карты маршрутизации в `_index_standards.md` при изменении файлов пакета обязательна (см. pre-commit и раздел «Автоактуализация индекса»).

## Автоактуализация индекса

Если пользователь тегнул этот файл (`@doc/standards/_index_standards.md`) без конкретной команды или задачи, агент выполняет автоактуализацию:

1. Прочитать все документы в `doc/standards/*.md` (кроме этого индекса)
2. Извлечь из каждого документа: имя, назначение, применимость, версию из заголовка
3. Обновить реестр документов: актуальные описания, применимость, триггеры
4. Обновить карту маршрутизации: актуальные темы задач и ссылки на разделы
5. Проверить единую версию пакета: все документы `doc/standards/*.md`, `AGENTS.md`, `CLAUDE.md`, `README.md` должны иметь одинаковую версию
6. Вывести отчет: какие документы обновлены, статус версии пакета, расхождения версий если есть
