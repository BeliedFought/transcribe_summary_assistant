# Навык: Проверка конфигурации Kilo

## Описание

Собирает и показывает актуальную конфигурацию Kilo: MCP-серверы (глобальные и проектные), правила доступа (permissions), файлы инструкций (AGENTS.md, CLAUDE.md, поле `instructions` в kilo.json). Позволяет убедиться что MCP работают и нужные правила загружаются агенту. При отсутствии необходимых MCP или правил -- добавляет их.

## Ожидаемые MCP-серверы

Список можно дополнять. MCP-серверы, которые должны быть в глобальном конфиге:

- `context7` -- `@upstash/context7-mcp` (документация библиотек)
- `github` -- `@modelcontextprotocol/server-github` (работа с GitHub)

## Ожидаемые правила сессии

Список можно дополнять. Правила, которые должны быть в глобальном конфиге в поле `instructions` или в отдельном файле инструкций:

- `~/.config/kilo/session_rules.md` -- агент перед началом каждой сессии читает `_index_standards.md` и применяет найденные стандарты; при работе с репо читает конкретные выдержки из `doc/standards/`, релевантные задаче, и руководствуется ими в работе
- В `session_rules.md` должно быть правило: требования к тексту (раздел 04.03 стандарта) применяются и в режиме планирования - план-файл создается по тем же правилам к тексту. Гарантирует соблюдение стиля текста в планах в любой сессии и в любом режиме

## Когда использовать

- Пользователь просит проверить конфигурацию Kilo, MCP-серверы, правила агента
- Пользователь хочет убедиться что MCP-серверы доступны и работают
- Пользователь спрашивает какие инструкции получает агент в начале сессии
- Пользователь хочет проверить что permissions настроены корректно
- Триггер-слова: go

## Предусловия

- Kilo установлен и доступен
- Есть доступ на чтение к `~/.config/kilo/` и корню проекта

## Инструкция

0. Если задача передается субагенту — субагент обязан до начала работы прочитать `doc/standards/_index_standards.md` (если проект содержит этот файл) и применять найденные стандарты в своей работе.

1. Прочитать глобальный конфиг `~/.config/kilo/kilo.json` и `~/.config/kilo/kilo.jsonc` — извлечь MCP-серверы, permissions, instructions.

2. Прочитать проектный конфиг `kilo.json` и `kilo.jsonc` (если есть) — извлечь MCP-серверы, permissions, instructions.

3. Прочитать все файлы инструкций, которые загружаются агенту:
   - `AGENTS.md` и `CLAUDE.md` в корне проекта
   - Файлы из поля `instructions` в `kilo.json`/`kilo.jsonc` (глобальном и проектном)
   - Глобальный `AGENTS.md`/`CLAUDE.md` из `~/.config/kilo/` (если есть)

4. Проверить наличие ожидаемых MCP-серверов из списка в разделе `Ожидаемые MCP-серверы`. Для каждого:
   - Если сервер присутствует в глобальном `kilo.json` -- проверить доступность (локальный: проверить команду, удаленный: проверить URL)
   - Если сервер отсутствует -- отметить в отчете как отсутствующий

5. Проверить наличие ожидаемых правил сессии из списка в разделе `Ожидаемые правила сессии`. Для каждого:
   - Если файл инструкции присутствует в поле `instructions` глобального `kilo.jsonc` -- прочитать его содержимое, убедиться что правила актуальны
   - Если файл инструкции отсутствует -- отметить в отчете как отсутствующий

6. Сформировать отчет и показать пользователю:
   - MCP-серверы: имя, тип, статус, отметка об отсутствующих
   - Правила сессии: список файлов инструкций, отметка об отсутствующих
   - Permissions: список разрешенных bash-команд, external_directory
   - Примечания: если есть расхождения или проблемы
   - В конце -- вопрос: какие из отсутствующих элементов добавить?

7. После получения согласия пользователя -- добавить только то, что он одобрил:
   - Отсутствующие MCP-серверы -- добавить в `~/.config/kilo/kilo.json`
   - Отсутствующие файлы правил -- создать, добавить путь в поле `instructions` глобального `kilo.jsonc`

## Критерии завершения

- Все конфиги прочитаны и разобраны
- MCP-серверы проверены на доступность
- Правила сессии проверены
- Отчет показан пользователю, получено согласие на добавления
- Отсутствующие элементы добавлены только после согласования
- Проблемы и предупреждения выделены отдельно

## Примеры

1. Пользователь: «проверь MCP» -> агент читает `~/.config/kilo/kilo.json` и проектный `kilo.json`, сверяется со списком ожидаемых MCP, выдает отчет со статусами, добавляет отсутствующие.
2. Пользователь: «настрой правила сессии» -> агент проверяет `instructions` в глобальном конфиге, создает `session_rules.md` если его нет, добавляет путь в конфиг.
3. Пользователь: «проверь конфигурацию» -> агент запускает полную проверку MCP + правил + permissions, выдает сводный отчет.

## Ограничения

- Никакие изменения не вносятся без явного согласия пользователя
- MCP-серверы только добавлять, не удалять существующие
- Не запускать MCP-серверы — только проверка их наличия и команд
- Правила сессии только добавлять, не удалять существующие инструкции
